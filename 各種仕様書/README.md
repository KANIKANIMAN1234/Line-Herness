# LINE Harness 各種仕様書（索引）

このフォルダには、**LINE Harness**（LINE 公式アカウント向けオープンソース CRM / マーケ自動化）の仕様を、**初心者でもイメージしやすい**ようにまとめた文書があります。

## 読み方の目安

| 順番 | 文書 | こんなときに読む |
|------|------|------------------|
| 1 | [01-システム仕様書](./01-システム仕様書.md) | 全体像・誰が何をするアプリか |
| 2 | [02-技術仕様書](./02-技術仕様書.md) | 使っている言語・サービス・フォルダ構成 |
| 3 | [08-インストール解説書](./08-インストール解説書.md) | まず手元／クラウドに載せたい |
| 4 | [03-DB仕様書](./03-DB仕様書.md) | データの保存のされ方 |
| 5 | [04-画面設計仕様書（フロントエンド）](./04-画面設計仕様書（フロントエンド）.md) | 管理画面の画面と操作の流れ |
| 6 | [05-バックエンド仕様書](./05-バックエンド仕様書.md) | API サーバー（Worker）の役割 |
| 7 | [07-API仕様書](./07-API仕様書.md) | 外部から REST で触るとき |
| 8 | [06-連携仕様書](./06-連携仕様書.md) | LINE・決済・Webhook など外部サービス |

## 用語メモ

- **管理画面**: ブラウザで開くダッシュボード（Vercel 等に配置する Next.js アプリ）。
- **API（Worker）**: Cloudflare Workers 上の Hono アプリ。管理画面・LINE・外部から呼ばれる。
- **EF**: 本プロジェクトの文脈では、**画面（エントリー／操作画面）＝フロントエンド**の設計として [04](./04-画面設計仕様書（フロントエンド）.md) にまとめています。

## ソースとの対応

- アプリ本体: リポジトリルートの `apps/`・`packages/`
- DB 定義: `packages/db/schema.sql` と `packages/db/migrations/`
- API 実装: `apps/worker/src/routes/`
- OpenAPI JSON: デプロイ後 `https://（WorkerのURL）/openapi.json`（[07](./07-API仕様書.md) 参照）

文書の更新日は Git の履歴を参照してください。
